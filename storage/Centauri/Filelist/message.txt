Ich will das Problem kurz umreißen:
Weil die Plugins, die ihr erstellt habt, die Parameter in der URL mitgeben, muss der Seiten Cache Mechanismus von Typo3 ausgeschaltet sein, damit die Teilnehmenden Einrichtungen sowie der Material Browser funktionieren.

Das wiederum bedeutet, dass die neu eingetragenen Inhalte nicht in den Suche Index aufgenommen werden.
Um dies zu tun, muss vom Admin der Seiten-Cache deaktiviert werden. Dann muss jede neue Seite einzeln von Hand aufgerufen werden um in den Index aufgenommen zu werden.
Danach muss der Admin den Cache wieder deaktivieren, damit Material Browser und Teilnehmende Einrichtungen wieder funktionieren.

Bei den Teilnehmenden Einrichtungen funktiniert das leider nicht. Hier werden in kurzen Abständen neue eingepflegt. Wenn ich dann eine Einrichtung suche, wird die im Suche-Ergebnis zwar angezeigt, der Link verweist dann aber auf eine Filterergebnisseite im bei den Teilnehmenden Einrichtungen auf der sich diese nicht befindet.
Wenn man z.B. in der Suche nach Ulm sucht, werden in der Suche Ergebnisseite Treffer für teilnehmende Einrichtungen in Ulm angezeigt. Wenn ich dann auf den Link klicke, komme ich auf eine Seite der Teilnehmenden Einrichtungen, auf der Ulm nicht auftaucht, sondern erst in der nächsten Ergebnisseite.
